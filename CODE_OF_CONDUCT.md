# Code of Conduct

## Our Pledge

We as contributors and maintainers pledge to make participation in the DevLens community a respectful, inclusive, and harassment-free experience for everyone.

We are committed to creating an open-source environment focused on:

- collaboration
- learning
- professionalism
- constructive communication

regardless of:

- experience level
- gender identity
- sexual orientation
- disability
- ethnicity
- religion
- nationality
- personal background

---

# Expected Behavior

Examples of behavior that contributes to a positive environment include:

- being respectful and constructive
- welcoming different viewpoints and experiences
- accepting feedback gracefully
- focusing on what is best for the community
- showing empathy toward other contributors
- helping create a beginner-friendly environment
- communicating professionally during reviews and discussions

---

# Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment or discrimination
- trolling or insulting comments
- personal attacks
- publishing private information without permission
- abusive or threatening behavior
- hate speech
- spam or intentionally disruptive actions
- repeated hostile behavior after warnings

---

# Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing community standards.

Maintainers may remove, edit, or reject comments, commits, issues, discussions, pull requests, or other contributions that violate this Code of Conduct.

Maintainers may also temporarily or permanently restrict participation for behavior considered inappropriate, threatening, offensive, or harmful to the community.

---

# Scope

This Code of Conduct applies within all DevLens community spaces, including:

- GitHub repositories
- issues
- pull requests
- discussions
- documentation
- community communication channels
- public project-related interactions

This Code of Conduct also applies when an individual is officially representing the DevLens project in public spaces.

---

# Reporting Issues

If you experience or witness unacceptable behavior, please report it privately to the project maintainer.

When reporting an issue, please include:

- relevant screenshots or links when available
- a short description of the incident
- any additional context that may help investigation

All reports will be reviewed responsibly and handled as privately as possible.

---

# Enforcement Guidelines

Possible enforcement actions may include:

- warning
- temporary restriction
- removal of content
- permanent ban from project spaces

The appropriate response will depend on the severity and frequency of the violation.

---

# Community Goals

DevLens aims to build a community that is:

- welcoming
- respectful
- developer-focused
- beginner-friendly
- professional
- collaborative

Healthy open-source communities help create better software for everyone.
